Updating the Git hooks

When you enter the development environment, our pre-commit.hook script is symlinked in your local git repository as .git/hooks/pre-commit.

Then when you make a git commit, the following happens:

  • The pre-commit.hook script runs the (confusingly named) pre-commit tool installed in the sandbox.

  • The pre-commit tool runs the hooks configured in .pre-commit-config.yaml

Sooner or later you'll want to update the pre-commit framework and the hooks.

How the hooks are executed

The .pre-commit-config.yaml file contains a list of "repos" and which of their "hooks" to be run. The repos are automatically downloaded by the pre-commit tool and the hooks are executed each in their own own virtualenv.

Notice there is also the local repo which contains the pylint hook. This results in pylint being executed directly. If pylint runs in a virtualenv it won't have access to the Python environment in our flatpak sandbox.

Funny the example in the local hooks documentation is about pylint.
